### STEP 1 - Analysis

- Analyze all changes made
- Based on the changes bump the version of the project if available
- Adjust the Readme.md file with the new changes - only if there is one.

### STEP 2 - Git operations

- Create a commit message using the Conventional Commits style.
- Ask the user to if the message looks good - Use the question tool.
- If they confirm - you can commit.
- Then ask the user if you can push the changes - Use the question tool - Options should be: Yes and No
